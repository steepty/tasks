# Конфигурация
+ env переменные, никакого хардкода
+ валидация при старте
+ разные конфиги для окружений ( если это требуется в задаче)

# Observability
+ structured logging  с request_id, user_id
+ Prometheus метрики — latency, error rate, очереди, бизнес-метрики
+ OpenTelemetry трейсинг сквозной
+ /healthz с проверкой всех зависимостей

# Надёжность
+ graceful shutdown
+ таймауты везде
+ retry с exponential backoff
+ circuit breaker для внешних сервисов

# База данных
+ миграции через golang-migrate/goose(у них есть разница, разработчик должен обосновывать выбор)
+ индексы на все поля фильтрации
+ pgx с connection pool(если постгрес)
+ `SELECT FOR UPDATE` где нужна блокировка

# Безопасность
+ пароли через bcrypt
+ JWT + refresh token
+ валидация входящих данных
+ параметризованные запросы

# Тестирование
+ юнит-тесты через моки
+ интеграционные тесты через testcontainers-go
+ тесты на корнер-кейсы 
+ `go test -race` обязательно
+ проверка линтером, конфиг в этой же папке(`.golangci.yml`)

# Docker
+ многоэтапный Dockerfile
+ docker-compose.yml одной командой
+ .env.example


# Саммари
+ Разработчик должен понимать смысл всех вышеперечисленных требований и делать их потому что он понимает для чего это нужно и в чем смысл, а не потому что я так написал
