# Задачи по синтаксису Go

То, что код не компилируется или падает с паникой, это во многих задачах ожидаемое поведение и задача состоит в том, чтобы это починить. Поэтому скорее всего не стоит искать проблему на своей стороне. 

Задачи запускаются по одной, цель ответить на все вопросы и/или починить код и заодно закрепить особенности языка.

Везде используется только стандартная библиотека, базовой установки го компилятора свежей версии будет достаточно для запуска и тестирования.

go.mod уже есть в корне, этого достаточно для запуска всех файлов.


Для усложения решения задач и эмуляции условий собеседования, предлагаю брать код из файла и запускать в https://go.dev/play/ или на другой платформе, так как IDE будет сразу подсвечивать проблемные места и описывать причину, но выбор остается за тобой.

На мой взгляд, наиболее удобный формат работы с задачами такой:
+ Открыть задачу
+ Без запуска попытаться объяснить как поведет себя код и ответить на вопросы
+ Запустить код
+ Исправить ошибку в коде или в своем ходе мыслей
+ Запустить повторно с исправлениями


Не вижу ничего плохого в использовании ИИ помощников и нейросетей в целом, смысл в понимании поведения кода и ответов на задачи, если все понятно и в будущем ошибок на подобных кейсах не будет, нет разницы каким образом был получен ответ
